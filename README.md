# BorderFlow 🚗🇺🇦

Интеллектуальная платформа мониторинга очередей на государственной границе Украины в режиме реального времени.

## Архитектура проекта
Проект состоит из трех основных частей:
- **Backend (NestJS)**: Обработка бизнес-логики, агрегация данных (Source Engine), REST API и Telegram-бот (grammY).
- **Frontend (Next.js 15)**: Пользовательский интерфейс с интерактивной картой (Leaflet), статистикой и админ-панелью.
- **Инфраструктура**: PostgreSQL (основная БД), Redis (для очередей BullMQ и кэша).

## Требования
- Node.js v18+
- Docker и Docker Compose (для запуска БД и Redis)

## Быстрый старт

### 1. Поднятие инфраструктуры (БД и Redis)
В корне проекта находится файл `docker-compose.yml`. Поднимите контейнеры:
```bash
docker compose up -d
```
> **Внимание:** Если у вас не установлен Docker, вам потребуется указать URL к внешней PostgreSQL базе данных в файле `backend/.env`.

### 2. Запуск Backend
Перейдите в папку `backend` и настройте окружение:
```bash
cd backend
npm install
```
Убедитесь, что в `backend/.env` прописан корректный `DATABASE_URL` и `TELEGRAM_BOT_TOKEN`.

Примените схему базы данных:
```bash
npx prisma db push
```

Запустите сервер:
```bash
npm run start:dev
```
Backend будет доступен по адресу: `http://localhost:3000/api`
Swagger документация API: `http://localhost:3000/api/docs`

### 3. Запуск Frontend
Перейдите в папку `frontend`:
```bash
cd frontend
npm install
npm run dev
```
Откройте браузер по адресу: `http://localhost:3001` (или 3000, если порт свободен).

## Основные модули
- **Source Engine**: Система автоматического сбора данных из разных источников. В данный момент реализована генерация мок-данных (`mockData()`) для демонстрации работы системы. Для подключения реальных парсеров (dpsu.gov.ua) необходимо реализовать логику парсинга HTML/API в `SourceEngineService`.
- **Telegram Bot**: Полноценный бот с использованием `grammY` и Inline-клавиатур.
